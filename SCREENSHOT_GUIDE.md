# App Store Screenshots - Generation Guide

## Screenshot Requirements

### iPhone Screenshots
- **Resolution:** 1170 × 2532 pixels (iPhone 14 Pro)
- **Format:** PNG or JPG
- **Count:** 3-5 screenshots recommended
- **Safe Area:** Content should fit within safe margins (not touching edges)

### iPad Screenshots (Optional but recommended)
- **Resolution:** 2048 × 2732 pixels (iPad Pro)
- **Format:** PNG or JPG
- **Count:** 1-3 screenshots

## How to Create Screenshots

### Option 1: Using Figma (Recommended)
1. **Create New File** in Figma
2. **Set Artboard Size:** 1170 × 2532 px (iPhone 14 Pro)
3. **Copy Elements from Landing Page:**
   - Hero section
   - Dashboard layouts
   - Feature cards
   - Color scheme (purple/yellow/white)
4. **Add Text Overlays:**
   - Main headline
   - Supporting copy
   - Call-to-action
5. **Export as PNG:**
   - Right-click artboard
   - "Export" → PNG format
   - Ensure 2x pixel density for sharpness

### Option 2: Screenshot from Running App
1. **Open App** on iPhone or iPhone simulator
2. **Navigate to Each Page:**
   - Landing page hero
   - Dashboard (if logged in)
   - Ritual/lesson page
3. **Take Screenshots:**
   - Hold power + volume up (iPhone)
   - Or use Xcode simulator screenshot tool
4. **Edit in Figma/Photoshop:**
   - Add text overlays
   - Add benefit callouts
   - Ensure readability

### Option 3: Design Tool (Sketch, Adobe XD)
1. **Create Artboard:** 1170 × 2532 px
2. **Use Design System:**
   - Colors: Purple (#2D1B69), Yellow (#FACC15), White, Black
   - Fonts: System fonts (SF Pro Display)
   - Components: Cards, buttons, badges
3. **Build Mockup:**
   - Add background images
   - Layer UI components
   - Add marketing copy
4. **Export:** PNG with high quality

## Design Guidelines for Screenshots

### Typography
```
Headline: 32-44px, Bold, White text
Subheading: 18-24px, Regular, White/Yellow
Body: 16-18px, Regular, White with 70% opacity
```

### Colors
```
Background: #0D0D0F (deep black/purple)
Accent: #FACC15 (yellow-400, CTAs)
Text Primary: #FFFFFF (white)
Text Secondary: rgba(255, 255, 255, 0.7)
```

### Spacing
```
Top Padding: 40px (safe from top notch)
Side Padding: 16px left/right
Bottom Padding: 40px (safe from bottom)
Element Spacing: 12-16px
```

### Safe Areas
- **Top:** 40px from top (notch clearance)
- **Bottom:** 40px from bottom (home indicator)
- **Left/Right:** 16px minimum padding

## Quality Checklist

Before uploading to App Store Connect:

- [ ] Resolution: 1170 × 2532 px (iPhone) or 2048 × 2732 px (iPad)
- [ ] Format: PNG or JPG (PNG recommended)
- [ ] Text is legible and not cut off
- [ ] Safe areas respected (no content in notch area)
- [ ] Colors match app design system
- [ ] No personal information visible
- [ ] Branding/logo present
- [ ] CTA is clear and prominent
- [ ] All 5 screenshots tell a story
- [ ] Metadata complete (each screenshot should be localized in all required languages)

---

## Upload to App Store Connect

1. **Sign in to App Store Connect**
2. **Navigate to:** My Apps > [App Name] > App Store > Screenshots
3. **Select Device Type:** iPhone 14 Pro (or other)
4. **Upload Screenshots in Order:**
   - Screenshot 1: Hero
   - Screenshot 2: Dashboard
   - Screenshot 3: Experience
   - Screenshot 4: Community (if available)
   - Screenshot 5: Progress (if available)
5. **Add Localized Metadata (Optional):**
   - Different screenshots for different languages
   - Localized text overlays
6. **Preview:** Use preview tool to verify appearance
7. **Submit:** Save and submit for review

## Pro Tips

### Do's ✅
- ✅ Show real app features and functionality
- ✅ Use contrasting colors for text readability
- ✅ Focus on benefits, not features
- ✅ Tell a story across all screenshots
- ✅ Use real testimonials/user quotes
- ✅ Highlight unique value proposition
- ✅ Make CTA clear and prominent
- ✅ Ensure text is large enough to read

### Don'ts ❌
- ❌ Use misleading or doctored screenshots
- ❌ Show features that don't exist in app
- ❌ Clutter with too much text
- ❌ Use only default/placeholder content
- ❌ Ignore safe areas (notch, home indicator)
- ❌ Use low-quality or blurry images
- ❌ Forget to test on actual devices
- ❌ Leave screenshots in another language
