# REPLIT AGENT PROMPT - MetaHers Voyages Platform

Build a complete Next.js 14 luxury tech education booking platform called “MetaHers Voyages” where women book intimate experiences (max 6 per event) to learn AI/Web3/Crypto in luxury settings in Newport Beach.

## Tech Stack

- Next.js 14 (App Router), TypeScript, Tailwind CSS, shadcn/ui
- Prisma ORM with PostgreSQL
- Stripe for payments
- NextAuth.js for authentication
- Resend/SendGrid for emails
- Mapbox GL JS for maps (or Leaflet as fallback)

## Design System & UX Principles

### Brand Identity

**This is a LUXURY LIFESTYLE BRAND, not a typical tech platform**

**Color Palette**:

- Primary: `#8B5CF6` (Purple - sophistication, transformation)
- Secondary: `#EC4899` (Pink - feminine power, energy)
- Accent: `#F59E0B` (Gold - premium, luxury)
- Neutral Light: `#FAFAFA` (backgrounds)
- Neutral Dark: `#1F2937` (text)
- Success: `#10B981` (confirmations)

**Typography**:

- Headings: `Playfair Display` (serif, elegant, luxury)
- Body: `Inter` (sans-serif, clean, readable)
- Use generous font sizes - nothing feels cramped or small

**Spacing**:

- Use generous white space
- Padding should feel luxurious, not cramped
- Mobile: minimum 16px padding on edges
- Desktop: minimum 24px padding, hero sections 60-80px

## Database Schema (Prisma)

```prisma
model User {
  id            String          @id @default(cuid())
  email         String          @unique
  name          String
  password      String
  createdAt     DateTime        @default(now())
  bookings      Booking[]
  questionnaires Questionnaire[]
}

model Voyage {
  id              String    @id @default(cuid())
  title           String
  slug            String    @unique
  category        String    // "AI", "Crypto", "Web3", "AI_Branding"
  sequenceNumber  Int       // 1-12
  description     String    @db.Text
  learningObjectives String[]
  date            DateTime
  time            String
  duration        String
  location        String
  venueType       String    // "Duffy_Boat", "Picnic", "Brunch"
  latitude        Float
  longitude       Float
  price           Int       // in cents
  maxCapacity     Int       @default(6)
  currentBookings Int       @default(0)
  included        String[]
  images          String[]
  status          String    @default("upcoming")
  bookings        Booking[]
  createdAt       DateTime  @default(now())
}

model Booking {
  id              String         @id @default(cuid())
  userId          String
  voyageId        String
  status          String         @default("confirmed")
  paymentStatus   String
  stripeSessionId String?        @unique
  amount          Int
  createdAt       DateTime       @default(now())
  user            User           @relation(fields: [userId], references: [id])
  voyage          Voyage         @relation(fields: [voyageId], references: [id])
  questionnaire   Questionnaire?
}

model Questionnaire {
  id                  String   @id @default(cuid())
  bookingId           String   @unique
  userId              String
  techComfortLevel    String
  specificTopics      String   @db.Text
  goals               String   @db.Text
  concerns            String?  @db.Text
  dietaryRestrictions String?
  howHeardAboutUs     String?
  completedAt         DateTime @default(now())
  booking             Booking  @relation(fields: [bookingId], references: [id])
  user                User     @relation(fields: [userId], references: [id])
}

model Waitlist {
  id        String   @id @default(cuid())
  email     String
  voyageId  String
  createdAt DateTime @default(now())
  notified  Boolean  @default(false)
}
```

- Skeleton screens (not spinners) for cards
- Smooth fade-in animations when content loads
- Loading overlay with brand colors for page transitions

### Error States

- Friendly error messages (never technical jargon)
- Suggestions for what to do next
- Contact support option
- Retry button

### Animations

- Smooth page transitions (fade)
- Card hover: lift + shadow increase
- Button hover: slight scale + shadow
- Modal: slide up + fade in background
- Progress bars: animated fill
- Success states: confetti or celebration animation
- All animations: 200-300ms duration

### Mobile Responsiveness

- Touch-friendly: minimum 44px touch targets
- Sticky CTAs on mobile (bottom bar)
- Collapsible sections to save space
- Swipeable carousels
- Mobile menu: slide-in from left
- Forms: full-width inputs, large labels

### Accessibility

- Proper heading hierarchy (h1 → h6)
- Alt text for all images
- Keyboard navigation support
- Focus states clearly visible
- ARIA labels for interactive elements
- Color contrast meets WCAG AA standards

## Critical Business Logic

### Booking Flow

1. User clicks “Book Now”
1. Check if logged in → if not, show login/signup modal
1. Check voyage capacity → if full, show waitlist option
1. Show booking review page
1. Create pending booking in database
1. Redirect to Stripe Checkout
1. On successful payment (webhook):
- Update booking to “confirmed”
- Decrement voyage.currentBookings
- Send confirmation email
- Redirect to onboarding

### Capacity Management

- Always check currentBookings < maxCapacity before allowing booking
- Use database transactions to prevent race conditions
- Show real-time availability
- When voyage fills up:
  - Update status to “full”
  - Show “Join Waitlist” instead of “Book Now”
  - Notify waitlist when spots open

### Email Triggers

- Booking confirmation (immediately after payment)
- Onboarding step 1 (1 hour after booking)
- Onboarding reminders (if not completed in 24 hours)
- Pre-voyage reminder (48 hours before)
- Post-voyage thank you (day after voyage)
- Questionnaire reminder (if not completed)

## API Endpoints

```typescript
// Public
GET  /api/voyages                    // List all voyages (with filters)
GET  /api/voyages/[slug]             // Single voyage details
POST /api/waitlist                   // Join waitlist

// Protected (authenticated)
POST /api/bookings                   // Create booking
GET  /api/bookings/user              // User's bookings
POST /api/questionnaires             // Submit questionnaire
GET  /api/dashboard                  // Dashboard data
POST /api/referrals                  // Generate referral

// Payment
POST /api/checkout/session           // Create Stripe session
POST /api/webhooks/stripe            // Handle webhooks

// Admin only
GET  /api/admin/voyages              // All voyages management
PATCH /api/admin/voyages/[id]        // Update voyage
GET  /api/admin/bookings             // All bookings
GET  /api/admin/questionnaires       // All responses
POST /api/admin/messages             // Send messages
GET  /api/admin/analytics            // Analytics data
```
