# Replit Agent Prompt: Human Design Chart Generator

## Project Overview
Build a Human Design chart generator API for a women's wellness app called "Metahers". The API should calculate and return Human Design charts based on birth data (date, time, location).

## Requirements

### Core Functionality
1. **Accept birth data input:**
   - Date of birth (YYYY-MM-DD)
   - Time of birth (HH:MM in 24-hour format)
   - Location (city, country or latitude/longitude)

2. **Calculate and return:**
   - Human Design Type (Manifestor, Generator, Manifesting Generator, Projector, Reflector)
   - Authority (Sacral, Emotional, Splenic, Ego, Self-Projected, Environmental, Lunar)
   - Profile (e.g., 1/3, 2/4, 3/5, etc.)
   - Defined/Undefined Centers (all 9 centers)
   - Channels and Gates
   - Incarnation Cross (if possible)

3. **Output format:** JSON response with all chart data

### Technical Requirements
- **Must be FREE and open source** - no paid API subscriptions
- Language: JavaScript/Node.js or Python preferred
- Should be self-hosted (no external API dependencies if possible)
- RESTful API endpoint structure
- Include error handling for invalid birth data
- Add basic documentation

## Implementation Options to Evaluate

Please research and choose the BEST free option from these:

### Option 1: hdkit (JavaScript)
- **GitHub:** https://github.com/jdempcy/hdkit
- **Language:** JavaScript/Node.js
- **License:** MIT (free)
- **Pros:** JavaScript-based, has sample apps, generates bodygraphs
- **Consider:** Setup complexity, documentation quality, maintenance status

### Option 2: SharpAstrology.HumanDesign (C#)
- **GitHub:** https://github.com/CReizner/SharpAstrology.HumanDesign
- **Language:** C#/.NET
- **License:** MIT (free)
- **Consider:** Would need .NET runtime, cross-platform compatibility

### Option 3: Build from Scratch
- Use ephemeris libraries (like Swiss Ephemeris) to calculate planetary positions
- Implement Human Design calculation logic manually
- **Libraries to consider:**
  - swisseph (Python or Node.js bindings)
  - astronomia (JavaScript)
  - Consider:** Development time vs using existing solution

## Evaluation Criteria

Choose the best option based on:
1. ✅ **Cost:** Must be 100% free (no API keys or subscriptions)

4. ✅ **Documentation:** Has clear usage instructions
5. ✅ **Maintenance:** Recently updated, not abandoned
